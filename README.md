# MBTI企業マッチングマップ(吹田みらい企業マッチングフェス用)

合同説明会(フェーズ1)で、**学生と企業をMBTI診断でマッチング**するWebアプリです。

- 企業側: 全社員にMBTI診断をしてもらい、タイプごとの人数を登録
- 学生側: 自分のMBTIを入力(わからなければアプリ内の簡易診断でOK)
- 学生には**会場ブースマップがマッチ度で色分け表示**され、企業説明・社員のMBTI分布・マッチ度・「話のきっかけ」が見られる

サーバー不要の静的Webアプリなので、GitHub Pages 等にそのまま公開できます。

---

## 画面構成

| ファイル | 対象 | 内容 |
|---|---|---|
| `index.html` | 学生 | MBTI入力(選択 or 簡易12問診断)→ 会場マップ/ランキング → 企業詳細 |
| `company.html` | 企業 | 会社情報+全社員のMBTI人数を入力 → 登録データ(JSON)を出力 |
| `admin.html` | 運営 | 企業データの取り込み・ブース配置・学生アプリ用データの書き出し |

## 当日までの運用フロー

1. **【事前】企業に依頼**: 全社員にMBTI診断(16personalities等の無料診断でOK)を受けてもらう
2. **【事前】企業が登録**: `company.html` で会社情報+タイプ別人数を入力し、生成されたJSONを運営へ送付
3. **【事前】運営が集約**: `admin.html` で各社のJSONを貼り付けて取り込み、実際の会場に合わせてブース配置(自動割り当てあり)→ `data.js` を書き出し
4. **【事前】反映**: 書き出した内容で `js/data.js` を置き換えて公開(GitHub Pagesなら push するだけ)
5. **【当日】学生が利用**: 受付でQRコードを配布 → 学生がスマホでMBTIを入力 → マップを見ながらブースを回遊

## 学生アプリの機能

- **16タイプ選択** または **簡易診断(12問・約1分)**。結果は端末に保存され再訪時も維持
- **会場マップ**: ブースがマッチ度で5段階に色分け(ベストマッチ/好相性/まずまず/ふつう/意外な出会い)、TOP3には順位バッジ
- **マッチ度ランキング**: 全社をマッチ度順にリスト表示
- **企業詳細**: 会社紹介・学生へのひとこと・社員のMBTI分布グラフ・「気が合いそうな社員の割合」・**ブースでの話のきっかけ3つ**

## マッチングの考え方

スライドの方針どおり「**MBTIは対話のきっかけであって採用判断ではない**」ため、説明可能なシンプルな方式にしています。

1. 学生のタイプと社員1人のタイプの相性 = 軸ごとの一致の加重和(最大100)
   - S/N(情報の受け取り方): 35点 — 会話の噛み合いに最も効くとされる軸
   - T/F(判断の基準): 25点
   - J/P(進め方): 20点
   - E/I(エネルギーの向き): 20点
2. 企業のマッチ度 = 社員のMBTI分布で重み付けした平均(を見やすくスケーリング)

ロジックは `js/matching.js` に集約しているので、重みの調整や方式変更も1ファイルで完結します。
アプリ内には「相性が低い企業との出会いにも価値がある」「採用の合否とは無関係」という注意書きを常に表示しています。

## 企業データの形式

```json
{
  "id": "c01",
  "name": "吹田精密工業",
  "industry": "製造業(精密部品)",
  "employees": 48,
  "description": "会社紹介文(学生向け)",
  "pr": "学生へのひとこと",
  "booth": { "label": "A-1", "x": 0, "y": 0 },
  "mbti": { "ISTJ": 9, "ISFJ": 5, "ESTJ": 6 }
}
```

- `mbti` はタイプごとの**人数集計のみ**(個人名は扱わないためプライバシーにも配慮)
- `booth.x` は列、`booth.y` は行(いずれも0始まり)。運営ページで自動割り当て可能

## オンライン集約(任意・GAS)

企業にJSONをメールで送ってもらう代わりに、Googleスプレッドシートへ直接集約することもできます。

1. `gas/Code.gs` の手順どおりにGASをウェブアプリとしてデプロイ
2. 発行URLを `js/config.js` の `GAS_ENDPOINT` に設定

これで `company.html` に「オンラインで送信」ボタンが現れ、学生アプリは開くたびに最新の企業データを取得します(取得失敗時は `data.js` にフォールバック)。

## 公開方法(GitHub Pages の例)

リポジトリの Settings → Pages でブランチを指定し、`/mbti-matching-app/index.html` のURLをQRコード化して当日配布してください。

## 動作確認

`index.html` をブラウザで開くだけで、サンプル12社(架空)のデータで動作を確認できます。
